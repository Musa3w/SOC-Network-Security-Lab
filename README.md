# 🛡️ Enterprise SOC & Network Security Architecture

## 📌 Project Overview
This repository documents the architecture, implementation, and analysis of an enterprise-grade Security Operations Center (SOC) and Network Security lab built entirely from scratch.
Running within an isolated, air-gapped VMware Workstation environment, this lab demonstrates enterprise network segmentation using OPNsense Firewall, centralized identity management through Windows Server Active Directory, and centralized log collection, ingestion, and analysis using Splunk Enterprise SIEM.

## 🏗️ Full Architecture & Workflow
The environment is designed around enterprise security principles, including network segmentation, centralized authentication, and centralized log management. OPNsense acts as the primary security gateway while Splunk provides complete visibility into security events across the environment.(architecture.png)

### 1. Infrastructure & Domain Backbone
The network is divided into dedicated security zones, with OPNsense serving as the central firewall and gateway. Windows Server manages the Active Directory infrastructure, allowing enterprise endpoints to authenticate through a centralized domain.

#### Step 1: Domain Controller Verification
The infrastructure is built on Active Directory Domain Services (AD DS), DNS, and DHCP running on Windows Server, successfully managing the APPLE.LOCAL domain.

![Domain Controller Verification](windows_server_dc.png)

#### Step 2: Endpoint Integration
The corporate workstation (CLIENT-10) has been successfully joined to the APPLE.LOCAL domain, providing centralized authentication and identity management.

![Endpoint Integration](windows_10_domain.png)

### 2. Log Management & Data Ingestion
To achieve centralized visibility, security logs generated by OPNsense and Windows systems are collected and ingested into Splunk Enterprise.

#### Step 1: Interactive Field Extraction
Raw Syslog events generated by OPNsense are parsed in Splunk using the Interactive Field Extractor (IFX) together with custom regular expressions to extract meaningful security fields.

![Interactive Field Extraction](splunk_field_extraction.png)

#### Step 2: Structured Security Fields
The log processing pipeline successfully normalizes raw log events into searchable security fields, including firewall actions (block), network interfaces (em1), TCP flags (tcp_flag=S), source IP addresses, destination ports, and protocol information.

![Structured Security Fields](splunk_parsed_fields.png)

### 3. Threat Simulation & Defensive Controls
To validate the security architecture, simulated attacks and reconnaissance traffic are generated from an external Kali Linux machine.

#### Step 1: Traffic Blocking Validation
An unauthorized connection attempt (curl -v) is initiated from the attacker machine (192.168.1.50). The firewall immediately blocks the request, resulting in a connection timeout.

![Traffic Blocking Validation](kali_attack.png)

#### Step 2: Live Firewall Monitoring
The blocked connection attempt is immediately recorded in the OPNsense Live View, demonstrating successful enforcement of perimeter security policies.

![Live Firewall Monitoring](opnsense_live_blocks.png)

#### Step 3: Firewall Policy Enforcement
Traffic filtering is enforced through a dedicated "Block Others from GUI" firewall rule configured on OPNsense.

![Firewall Policy Enforcement](opnsense_firewall_rules.png)

### 4. Security Monitoring Dashboard
All security events collected from the infrastructure are centralized within Splunk Enterprise and visualized through an enterprise security dashboard.

The dashboard provides actionable visibility for Blue Team operations:
* **Total Blocked Attacks:** Displays the total number of blocked firewall events (214).
* **Top 5 Attacker IP Addresses:** Identifies the most active source IP addresses generating blocked traffic.
* **Blocked Attacks Over Time:** Visualizes attack frequency and trends across the monitored time period.

![Security Monitoring Dashboard](enterprise_soc_dashboard.png)

## 🚀 Deployment Guide
To recreate this lab environment:
1. Configure an isolated Host-Only network using VMware Workstation.
2. Deploy OPNsense with separate WAN and LAN interfaces.
3. Install Windows Server and promote it to a Domain Controller for the APPLE.LOCAL domain.
4. Join Windows 10 clients to the Active Directory domain.
5. Install Splunk Enterprise and configure a Syslog receiver.
6. Forward OPNsense Syslog events to Splunk.
7. Create custom field extractions for firewall events (e.g., src_ip, dest_port, tcp_flag).
8. Generate test traffic from Kali Linux and verify that security events are captured in both OPNsense and Splunk dashboards.
